# Portfolio

## Documentation portfolio
Below is a small selection of documentation sites delivered in whole or in part by authors at Technically Write Ltd. These examples reflect the range of platforms and publishing workflows we support across developer documentation, product documentation, and API reference. Keep in mind that very often, Technically Write Ltd. must work on some customer projects with the tools and structure already in place. 

:::note
Only a limited number of examples can be shared publicly. Much of our work is covered by client Intellectual Property restrictions and/or security requirements. We will always share what we can, but customer confidentiality is priority.
:::

**[Technically Write Ltd.](https://www.technicallywrite.co.uk/)**  
**Stack:** Docusaurus + plugins  

The Technically Write Ltd website, implemented as a modern documentation-style site using Docusaurus. Built to support structured content, scalable navigation, and a maintainable authoring workflow.

---

**[Blackwall Documentation](https://docs.blackwall.com/)**     
**Stack:** MkDocs + plugins  

Documentation for Blackwall’s platform, covering core product concepts and practical guidance to help users deploy, configure, and operate the service effectively. Published using MkDocs and structured for clear navigation across user and operational workflows. The doc site is currently being restructured, rebranded, and redesigned.

---

**[R3 Documentation (Corda)](https://docs.r3.com/?utm_source=chatgpt.com)**    
**Stack:** Hugo + plugins + tools  

R3 develops Corda, an enterprise distributed ledger platform used to build and run interoperable business networks. The documentation covers platform concepts, operations, developer guidance, and reference material, and is published using Hugo.

---

**[Trilio (for Kubernetes)](https://docs.trilio.io/kubernetes)**    
**Stack:** GitBook  

Documentation for Trilio’s Kubernetes-focused data protection and backup solution. Contributions were made across the wider doc set during the engagement, with the majority focused on Kubernetes content.

---

**[WhiteHat Security / Black Duck — Continuous Dynamic Docs](https://source.whitehatsec.com/help/sentinel/api/index.html)**  
**Stack:** Antora + tools  

Product documentation for the Continuous Dynamic platform, supporting dynamic application security testing to help teams identify vulnerabilities in web applications. WhiteHat Security is now part of Black Duck.

---

**[WhiteHat Security / Black Duck — API Documentation](https://apidocs.whitehatsec.com/whs)**    
**Stack:** ReadMe (readme.io)  

API reference and developer guidance for the Continuous Dynamic API suite. During this work, the API documentation was migrated from an Antora-hosted approach to a ReadMe-based experience, improving usability and customer satisfaction.
